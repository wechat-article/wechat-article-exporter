# 本地部署与使用指南

本文档记录在 Windows 原生环境（非 WSL）下通过源码部署 wechat-article-exporter 的完整流程，以及核心功能的使用说明。

---

## 1. 环境要求

| 工具 | 最低版本 | 用途 | 验证命令 |
|------|---------|------|----------|
| Node.js | >= 22 | 运行时 | `node -v` |
| corepack | 内置于 Node 16.9+ | 管理 yarn 版本 | `corepack -v` |
| yarn | 1.22.22 | 包管理器 | `yarn -v` |
| Git | 任意 | 克隆源码 | `git --version` |

---

## 2. Windows 环境准备

### 2.1 安装 Node.js

前往 https://nodejs.org 下载 LTS 版本安装包（.msi），安装时勾选"Add to PATH"。

### 2.2 启用 corepack 与 yarn

以 **管理员权限** 打开 PowerShell 或 CMD，执行：

```powershell
corepack enable
corepack prepare yarn@1.22.22 --activate
```

> 为什么需要管理员权限？corepack 需要在 Node 安装目录下创建 yarn 的 shim 脚本，普通权限无写入权限。

### 2.3 验证

```bash
node -v          # 应输出 v22.x.x
corepack -v      # 应输出版本号
yarn -v          # 应输出 1.22.22
```

三行都通过即可继续。

---

## 3. 克隆与安装

```bash
# 克隆源码
git clone https://github.com/wechat-article/wechat-article-exporter.git
cd wechat-article-exporter

# 安装依赖（仅安装在项目 node_modules/，不污染全局）
yarn install
```

---

## 4. 环境变量配置

```bash
cp .env.example .env
```

`.env` 完整说明：

| 变量 | 必填 | 说明 | 推荐值 |
|------|:----:|------|--------|
| `NITRO_KV_DRIVER` | 是 | KV 存储驱动。`fs` 表示文件系统持久化，`memory` 为内存（重启丢失） | `fs` |
| `NITRO_KV_BASE` | 是 | KV 数据存储路径（仅在 `fs` 驱动下生效） | `.data/kv` |
| `NUXT_DEBUG_MP_REQUEST` | 否 | 调试微信代理请求日志，仅开发环境生效 | `false` |
| `NUXT_AGGRID_LICENSE` | 否 | AG Grid 企业版授权密钥，不填则数据表格有水印提示 | （留空） |
| `DEBUG_KEY` | 否 | 调试端点认证密钥 | （留空） |

> **关键**：`NITRO_KV_DRIVER=fs` 必须设置。默认值 `memory` 会导致服务重启后登录会话丢失。

## 5. 启动与运行

### 5.1 开发模式

```bash
yarn dev
```

访问 `http://localhost:3000`。开发模式支持热更新，修改代码后自动刷新。

### 5.2 生产模式

```bash
# 构建
yarn build

# 运行
NODE_ENV=production HOST=0.0.0.0 PORT=3000 node .output/server/index.mjs
```

### 5.3 Docker 方式

```bash
docker pull ghcr.io/wechat-article/wechat-article-exporter:2.3.16
docker run -d -p 3000:3000 ghcr.io/wechat-article/wechat-article-exporter:2.3.16
```

> Docker 镜像自带 Chromium，支持 PDF 导出。源码部署默认不支持 PDF 导出。

## 6. 首次使用流程

### 6.1 登录

1. 打开 `http://localhost:3000`，页面自动跳转到控制台
2. 首次进入会弹出微信扫码登录窗口
3. 使用**微信公众平台管理员**的微信扫码登录
4. 登录成功后，会话有效期约 4 天，过期需重新扫码

> 注意：需要有一个微信公众号的管理员权限。项目利用公众号后台"写文章时搜索其他公众号文章"的接口来抓取数据。

### 6.2 搜索并添加公众号

1. 进入"公众号管理"页面
2. 点击"添加"按钮
3. 在搜索框输入公众号名称关键词
4. 从搜索结果中选择目标公众号
5. 选中后系统自动开始同步该公众号的第一页文章

### 6.3 同步文章

1. 在公众号列表中，点击某个公众号的同步按钮
2. 系统自动分页拉取该公众号的所有文章
3. 文章元数据（标题、摘要、发布时间等）缓存到浏览器 IndexedDB
4. 可在"设置"中配置同步的日期范围

### 6.4 下载与导出

1. 进入"文章下载"页面，选择一个公众号
2. 文章列表以表格形式展示，支持按标题、作者、时间、原创标签等条件筛选
3. 选择需要导出的文章，点击导出按钮
4. 选择导出格式

## 7. 核心功能说明

### 7.1 按公众号抓取（核心功能）

这是项目的核心功能。通过微信公众平台接口 `searchbiz` 搜索公众号，再通过 `appmsgpublish` 接口按公众号的 `fakeid` 分页拉取文章列表，实现抓取指定公众号所有文章。

- 搜索支持关键词模糊匹配
- 同步后文章列表缓存在浏览器本地，避免重复请求
- 支持增量同步（只拉取新文章）

### 7.2 多格式导出

| 格式 | 是否需要先下载内容 | 说明 |
|------|:-----------------:|------|
| JSON | 否 | 原始文章数据 |
| Excel | 否 | 使用 ExcelJS 生成表格 |
| HTML | 是 | 100% 还原文章排版与样式，图片内嵌打包 |
| TXT | 是 | 纯文本提取 |
| Markdown | 是 | 使用 Turndown 将 HTML 转 Markdown |
| Word | 是 | 内测中 |
| PDF | 是 | 需要服务端 Puppeteer/Chromium 支持，源码部署默认不可用 |

> "需要先下载内容"的格式，需先在文章列表中点击下载按钮获取文章 HTML 正文，然后才能导出。

### 7.3 文章过滤

支持多维度筛选：
- **文本过滤**：标题、摘要、文章 ID、链接
- **日期过滤**：创建时间、更新时间
- **数值过滤**：阅读量、点赞数、分享数、评论数
- **集合过滤**：作者、是否原创、是否删除、内容/评论下载状态、文章类型

### 7.4 批量下载与合集下载

- **批量下载**：在文章列表中多选文章，一键批量下载内容
- **合集下载**：进入"合集下载"页面，按公众号的文章合集批量抓取
- **单篇下载**：进入"单篇文章下载"页面，粘贴微信文章链接直接下载

### 7.5 评论与阅读量

导出评论、阅读量、分享数等数据需要额外的 Credentials（微信客户端接口参数），需通过抓包获取：

- 在微信客户端打开目标公众号文章
- 使用抓包工具获取 `__biz`、`uin`、`key`、`pass_ticket` 等参数
- 在设置页面填入 Credentials

### 7.6 数据存储

- **客户端**：IndexedDB（Dexie），数据存储在浏览器中，清除浏览器数据会丢失
- **服务端**：KV 存储（文件系统或内存），保存登录会话 Cookie

## 8. 已知限制与注意事项

| 限制 | 说明 |
|------|------|
| 需要公众号管理员账号 | 必须有微信公众号的管理员权限才能扫码登录 |
| 登录会话有效期 | 约 4 天，过期后需重新扫码 |
| PDF 导出不可用 | 源码部署缺少 Chromium，如需 PDF 请用 Docker 部署 |
| 阅读量/评论需抓包 | 这些数据使用微信客户端接口，需额外获取 Credentials |
| 公共代理有请求限制 | 项目提供的公共代理节点有每日请求限额，私有部署建议配置自己的代理 |
| 反爬机制 | 请求频率过高可能触发微信风控，建议控制并发和频率 |

## 9. 常见问题

### Q: 登录后提示"session expired"怎么办？

微信公众平台的登录会话有有效期，过期后重新扫码登录即可。

### Q: 搜索公众号搜不到？

- 确认关键词是否正确
- 微信接口的搜索结果有数量限制，尝试使用更精确的公众号名称
- 检查网络连接和代理配置

### Q: 文章同步不完整？

- 可能是请求频率过高被限流，稍后重试
- 在设置中调整同步日期范围
- 清除浏览器缓存后重新同步

### Q: 如何配置代理？

进入"设置"页面，在代理配置中填入自定义代理节点地址。私有部署默认走服务端代理，通常不需要额外配置。

### Q: .data/kv 目录在哪里？

在项目根目录下。如果使用 `yarn dev` 启动，该目录会在项目根目录自动创建。

### Q: 依赖安装报错？

- 确认 Node 版本 >= 22：`node -v`
- 确认 yarn 版本：`yarn -v`
- 尝试删除 `node_modules` 和 `yarn.lock` 后重新 `yarn install`
- 网络问题可配置 npm 镜像：`yarn config set registry https://registry.npmmirror.com`
